# 命令说明
```
##################### 创建补丁 #####################
# 适用于未放入暂存区stash的情况，但是新增的代码无法比较
git diff > test.patch

# 推荐，使用提交的代码打补丁
# -n, 最近的n次提交
git format-patch -1

# 生成a1b2c3d之后所有的提交
git format-patch a1b2c3d

# git format-patch <commit_id_1>..<commit_id_2>
# 生成commit a1b2c3d 到 e4f5g6h 之间的补丁
git format-patch a1b2c3d..e4f5g6h

# 将生成的补丁保存到指定目录
# git format-patch -o <output_directory> <commit_id_1>..<commit_id_2>
git format-patch -o patches a1b2c3d..e4f5g6h


##################### 打补丁 #####################
# 测试补丁是否能应用成功
git apply --check test.patch

# 一次打一个补丁, 需要接着执行 git add . 和 git commit
git apply test.patch

# 可以一次打多个补丁, 会直接提交
# git am *.patch
```

# 使用
```shell
# 最近3次提交，保存到指定文件夹
git format-patch -3 -o ~/Desktop/patch

# 放弃调以前的am信息，否则会报错
git am --abort
# git am 可以一次合并一个文件 或者 一个目录
git am patch/*.patch
```
